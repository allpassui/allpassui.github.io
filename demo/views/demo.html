CDN

在页面导入Js和Css文件,先引用ap_page.js,然后vue.js、ap_ui.js,最后UI组件的index.js

<script src="../../ap_core/ap_page.js"></script>
<script src="../../ap_scripts/basic/vue.js"></script>
<script src="../../ap_core/ap_ui.js"></script>
<script src="../../dist/index.js"></script>

如何使用

前台页面中定义<ap-input id="a"></ap-input>这样的html。

后台ts中如这样var stu = new ap.module.student.stumanage("../workers/studentbll.js");传入相应的bll即Webworker路径。

在页面加载的4个不同阶段进行各种不同的操作，复写相应的4个函数。

initAtInteractive的时候进行页面组件的注册，this.txtSearch = new ap.ui.widget.input("txtSearch");该状态在页面加载的最开始，所以能够将每个自定义组件的UI渲染出来，不会出现白屏现象。

renderData 在该状态下进行静态数据的绑定。

renderDataAsync 在该状态下进行动态数据的绑定，与webworker进行数据的交互。

initAtComplete 在该状态下页面元素全部渲染完毕，相当于window.onload()。

注意事项

1、页面的ts构造函数中，第一个参数是页面bll路径，第二个参数可传可不传，是页面一开始调用的action,action具体是什么下面做说明。

2、action是封装的一个方法，可以与webworker进行数据交互，以/{a}/{b}的形式传递。a指的是页面的方法名，b可以是页面的对象也可以是string，当b是页面对象时 isVue: true, refid: "txtSearch", key: "value" ,按照这样的形式传递的话，可以动态的取到vue组件的值。当然也可以不是Vue的组件，可以定义你自己需要的变量。

3、所有的页面与webworker的通信都按照一个定好的协议执行。export interface IWorker {fn?: any,args?: any,callback?:any,data?:any} fn表示页面执行的方法名,args表示页面往webworker传递的参数，callback表示webworker回调的函数值,data表示回调的数据。

4、webworker自定义导入的js在构造函数中导入，arr是自定义导入的js，pathlevel是目录层级，0指的是和ap_scripts同层级，pathlevel递增表示目录层级不断递增。

5、框架用的ajax库默认是axios库，这个库在ie上使用要es6-promise.auto.js这个库,axios都是异步的，同步操作推荐使用promise。

6、默认的一些配置在acbb_basicconfig.acg中。
